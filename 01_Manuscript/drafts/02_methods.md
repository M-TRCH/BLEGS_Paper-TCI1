## 2.1 แพลตฟอร์มหุ่นยนต์ BLEGS

### 2.1.1 โครงสร้างเชิงกลและกลไก

### 2.1.2 ระบบขับเคลื่อนและเซนเซอร์

หุ่นยนต์ BLEGS ขับเคลื่อนด้วยมอเตอร์ BLDC จำนวน 8 ตัว (ขาละ 2 ตัว) ควบคุมด้วยเทคนิค Field-Oriented Control (FOC) ร่วมกับ Space Vector Pulse Width Modulation (SVPWM) โดยใช้ไมโครคอนโทรลเลอร์ STM32G431CBU6 (ARM Cortex-M4, 170 MHz) เป็นตัวควบคุมประจำมอเตอร์แต่ละตัว ระบบส่งกำลังใช้เฟืองทดอัตราส่วน 8:1

### 2.1.3 ระบบสื่อสาร

การสื่อสารระหว่างคอมพิวเตอร์ควบคุมส่วนกลางกับไมโครคอนโทรลเลอร์ประจำมอเตอร์ใช้ Binary Protocol v1.2 ผ่านช่องทาง UART ด้วยอัตราบอด (Baud Rate) 921,600 บิตต่อวินาที มอเตอร์ทั้ง 8 ตัวถูกกำหนดชื่อตามตำแหน่งของขาและแกนหมุน ดังนี้: FL_A, FL_B (ขาหน้าซ้าย), FR_A, FR_B (ขาหน้าขวา), RL_A, RL_B (ขาหลังซ้าย) และ RR_A, RR_B (ขาหลังขวา)

โปรโตคอลรองรับการส่งคำสั่งตำแหน่งเป้าหมาย (Position Setpoint) และรับข้อมูลตำแหน่งจริง (Actual Position) กลับมาในรอบเดียวกัน ทำให้สามารถบันทึกข้อมูล Setpoint กับ Actual พร้อมกันได้

## 2.3 การวัดตำแหน่งเชิงภาพ (Vision-based Position Estimation)

### 2.3.1 การติดตั้ง ArUco Marker

เพื่อวัดตำแหน่งจริงของข้อต่อบนกลไก ได้ติดตั้ง ArUco Marker (Dictionary: DICT_6X6_250, ขนาด 31.25 มม.) บนจุดข้อต่อของกลไก Five-Bar Linkage จำนวน 4 จุด โดยกำหนด ID ดังนี้:

- ID 1 = จุด A (จุดหมุนมอเตอร์ตัวที่ 1)
- ID 2 = จุด B (จุดหมุนมอเตอร์ตัวที่ 2)
- ID 3 = จุด C (ปลายลิงก์ AC)
- ID 4 = จุด D (ปลายลิงก์ BD)

### 2.3.2 กล้องและการสอบเทียบ

### 2.3.3 การคำนวณตำแหน่งปลายขา (จุด E)

ตำแหน่งของจุด A, B, C, D ได้จากจุดกึ่งกลางของ ArUco Marker ที่ตรวจจับได้ในแต่ละเฟรม ตำแหน่งปลายขา (จุด E) คำนวณในปริภูมิพิกเซล (Pixel Space) ด้วยวิธี Circle Intersection ตามขั้นตอนดังนี้:

1. วัดระยะทาง AC และ BD ในหน่วยพิกเซลจากตำแหน่ง Marker ที่ตรวจจับได้
2. คำนวณรัศมีวงกลมรอบจุด C และ D โดยใช้อัตราส่วนความยาวลิงก์:

$$r_{CE} = d_{AC} \times \frac{L_{CE}}{L_{AC}} = d_{AC} \times \frac{145}{105}$$

$$r_{DE} = d_{BD} \times \frac{L_{DE}}{L_{BD}} = d_{BD} \times \frac{145}{105}$$

3. หาจุดตัดของวงกลมสองวง (Center C, radius $r_{CE}$) กับ (Center D, radius $r_{DE}$)
4. เลือกจุดตัดที่อยู่ฝั่งปลายขา (ด้านล่าง) เป็นตำแหน่งจุด E

### 2.3.4 การแปลงพิกัดจากพิกเซลเป็นหน่วยมิลลิเมตร

เพื่อแปลงพิกัดจากปริภูมิพิกเซลเป็นระบบพิกัดของหุ่นยนต์ (Robot Frame) ในหน่วยมิลลิเมตร ใช้จุด A และ B เป็นจุดอ้างอิง เนื่องจากทราบระยะทางจริงระหว่าง A กับ B คือ 85 มม. (Motor Spacing) สเกลแปลงหน่วย (Scale Factor) คำนวณจาก:

$$\text{scale} = \frac{85 \text{ mm}}{d_{AB,\text{px}}}$$

โดย $d_{AB,\text{px}}$ คือระยะทางระหว่างจุด A กับ B ในหน่วยพิกเซล จากนั้นแปลงพิกัดของจุดทุกจุดเป็นหน่วยมิลลิเมตร โดยใช้จุดกึ่งกลาง AB เป็นจุดกำเนิด

ข้อมูลตำแหน่งปลายขา ($E_x$, $E_y$) ในหน่วยมิลลิเมตรถูกบันทึกลงไฟล์ CSV พร้อม Timestamp ของแต่ละเฟรม

## 2.4 แนวทางชดเชยความคลาดเคลื่อน (Data-Driven Compensation)

### 2.4.1 การเปรียบเทียบตำแหน่ง Encoder กับ Vision

ตำแหน่งปลายขาจากเซนเซอร์วัดมุมคำนวณผ่านจลนศาสตร์ไปข้างหน้า (Forward Kinematics) ของกลไก Five-Bar Linkage:

$$\vec{P}_{E,\text{enc}} = \text{FK}(\theta_1, \theta_2)$$

โดย $\theta_1$ และ $\theta_2$ คือมุมข้อต่อจาก Encoder ตำแหน่งนี้ถูกเปรียบเทียบกับตำแหน่งจริงที่วัดจาก Vision:

$$\vec{P}_{E,\text{vis}} = [E_{x,\text{vis}}, E_{y,\text{vis}}]$$

ค่าความคลาดเคลื่อนเชิงตำแหน่งคือ:

$$\vec{\epsilon} = \vec{P}_{E,\text{vis}} - \vec{P}_{E,\text{enc}}$$

### 2.4.3 ตัวชี้วัดผล

ใช้ตัวชี้วัดต่อไปนี้ในการประเมินผลก่อนและหลังชดเชย:

- **RMSE ตำแหน่งปลายขา** (แยกแกน X, Y และรวม):

$$\text{RMSE} = \sqrt{\frac{1}{N}\sum_{i=1}^{N}(\hat{P}_i - P_{i,\text{ref}})^2}$$

- **RMSE มุมมอเตอร์** (Setpoint vs Actual):

$$\text{RMSE}_\theta = \sqrt{\frac{1}{N}\sum_{i=1}^{N}(\theta_{i,\text{actual}} - \theta_{i,\text{setpoint}})^2}$$

- **ความถี่หลัก** (Dominant Frequency) จากการวิเคราะห์ FFT ของสัญญาณมุมมอเตอร์
- **ค่าผิดพลาดสูงสุด** (Maximum Absolute Error)
